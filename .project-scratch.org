# Action Classification
To evaluate action classification performance, both the base models and their PoseOFF-augmented variants (Figure 3) were trained and tested on full input sequences
No temporal masking was applied, simulating an offline action recognition setting.

The classification results are reported in Table I, with the best-performing model for each dataset and evaluation highlighted.
Across all architectures, incorporating PoseOFF embeddings yields average accuracy improvements of 2.14%, 6.84%, and 11.22% on NTU RGB+D 60, NTU RGB+D 120, and UCF101, respectively.
These gains indicate that PoseOFF embeddings capture informative cues about underlying human motion.

The improvements on UCF101 are particularly notable, with an average accuracy increase of 11.22%.
As UCF101 consists of RGB-only videos captured “in the wild,” without depth or skeleton annotations, this result suggests that PoseOFF embeddings provide salient motion information well suited to real-world scenarios.

# Action Anticipation
To assess whether early visual motion cues extracted via PoseOFF enable earlier action prediction at comparable accuracy to baseline models, temporal masking was used to simulate partial observation of ongoing actions.
The MS-G3D and ST-GCN++ models were trained using full sequences (100% observation ratio) and evaluated at 10% observation ratio increments.
InfoGCN++ incorporates temporal masking natively and produces frame-level predictions over a fixed 64-frame input, making interval-based evaluation unnecessary [8].
Action anticipation results are shown in Figure 4, using the Cross-Subject (CS) evaluation protocol on NTU RGB+D 60 (top) and the second evaluation protocol defined for UCF101 (bottom).

Figure 4 reports average performance as models observe increasing portions of each sequence.
Each point on the blue and orange curves represents the classification accuracy of the baseline and PoseOFF-augmented models, respectively, at a given observation ratio.
The gray dotted line indicates the baseline model’s peak performance under full observation, allowing direct comparison with PoseOFF-embedded variants.
The intersection between the gray and blue curves denotes the observation ratio at which the PoseOFF-embedded model matches the baseline’s full-sequence accuracy.

The results are the average performance of the model’s as they “see” more of each action sequence. In Figure 4, each point along the blue and orange curves corresponds to the action classification accuracy (y-axis) of the baseline and PoseOFF-embedded models, respectively, for a given observation ratio (x-axis). The gray dotted lines in these graphs indicates the highest performance of the base model, in order to compare it’s performance with the PoseOFF embedded versions of the models. The x-value at which the gray and blue lines intersect denotes the observation ratio at which the PoseOFF embedded model achieves the same accuracy as the base model with full observation.

The action anticipation results on the NTU RGB+D 60 dataset, shown in the first row of Figure 4, show that all models are able to predict the action taking place with fewer frames. For the ST-GCN++ and MS-G3D models, the PoseOFF embedded version of the model only needs to observe 80% of the entire action sequence to achieve the same performance as the base model at 100% observation ratio. For InfoGCN++, which was specifically designed for the task of action anticipation, the PoseOFF embedded model needs to only see half of the input sequence to achieve the same performance as the base model at 100% observation ratio.
The action anticipation results on the UCF101 dataset, shown in Figure 4, show again that PoseOFF embedded models require fewer frames to achieve the same action classification accuracy as the base model with a full observation of the sequence. For all models, PoseOFF features enable the model to predict the action with the same accuracy as the base model after only seeing half of the full sequence.

# Performance and Implementation Considerations
PoseOFF sampling provides a considerably more data efficient method of embedding optical flow information than training a neural network to learn from the entire flow field. Table II provides a comparison of data sizes required for pose only, pose with full optical flow, and PoseOFF for a single video containing 100 frames at a resolution of 1920 × 1080 pixels.
The changes required to adapt all three models were minimal and confined to only the first layers of each network through the PoseOFF embedding layer depicted in Figure 3. This embedding layer introduces a comparatively small amount of additional trainable parameters, and some computational overhead. A comparison of the number of trainable model parameters and a measurement of the asynchronous computation time for the base models and their PoseOFF embedded counterparts are presented in Table III. A higher number of trainable parameters generally corresponds to longer training times. Inference time was calculated by measuring the average time each model took to compute and predict a class label for 1000 random samples.
The results presented in Table III highlight the relatively minor impact in terms of model size (number of model parameters) and inference speed. The introduction of the PoseOFF embedding layer increased model size by an average of 1.88% by when compared to the number of trainable parameters of the base model Inference speed increased by an average of 0.03ms or 5.77% with respect to the base model. The smallest model, ST-GCN++ saw the greatest increase in both the number of model parameters and inference time. Given the speed at which the base models already operate, this increase is negligible.
